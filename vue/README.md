# Frontend (Vue.js) – Setup Guide

## Prerequisites
- Node.js (recommended latest LTS version)  
- npm (comes with Node.js)  

## Environment Setup
1. Install project dependencies:  
   ```bash
   npm install
   ```
2. Copy the environment configuration file and adjust it:  
   ```bash
   cp .env.example .env
   ```
3. In the `.env` file, set the `PROXY_URL` variable:
   - It should point to your backend (API) instance.  
   - Example:  
     ```
     PROXY_URL=http://your_instance.local
     ```

## Run Modes

### 1. Development (standard) – use this when you want to connect the frontend to a remote backend
Run the Vue app with the built-in Vite dev server:  
```bash
npm run dev
```
- The project will be available at the address shown in the console (usually `http://localhost:5173`).  

### 2. Development – use this when you want to develop the frontend on the same local server as your backend
There’s an alternative dev mode when working with a MintHCM Docker instance:  

1. In `.env`, set `PROXY_URL` to point to the internal backend address **without the port** (it defaults to port 80).  
   - Example:  
     ```
     PROXY_URL=http://backend
     ```
2. Start the dev server with:  
   ```bash
   npx vite --host 0.0.0.0
   ```
3. Open the URL generated by Vite in your browser (usually `http://localhost:5173`).  

### 3. Production Build
To create a production-ready build:  
```bash
npm run build
```
The output will be generated in the `dist/` directory, ready to be deployed to a server.
